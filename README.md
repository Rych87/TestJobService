Readme

Windows-приложение System Monitor Agent. Собирает информацию о машине и отправляет её на HTTP API. Может работать как служба Windows, так и как консольное приложение. 

Для сборки приложения нужно собрать решение в Visual Studio в конфигурации Release либо командой ``` dotnet build -c Release ```. После сборки в выходной папке находятся файлы приложения, а так же два скрипта powershell позволяющие установить(`install.ps1`) / удалить(`uninstall.ps1`) приложение в качестве службы Windows. Скрипты важно выполнять от имени администратора. Запускаем Powershell от имени администратора, переходим в папку приложения, выполняем команду `.\install.ps1`. Служба автоматически запускается при установке. Остановить или повторно запустить можно через управление службами Windows. 

Настройки приложения содержатся в файле `appsettings.json`, в секции `Settings` содержится адрес на который уходит запрос, интервал замера, таймаут запроса, список отслеживаемых процессов и путь логов.
пример файла конфигурации:
```  
"Settings": {
    "ApiUri": "http://localhost:7073/testapi2",
    "Interval": 30,
    "TimeOut": 30,
    "TrackedProcesses": ["Shellhost"],
    "logPath": "logs/"
  }
```
Перед запуском важно установить адрес отправки запросов. Конфигурацию можно менять на лету, не останавливая службу.

В качестве http post запросов приложение отправляет json содержащий всю информацию согласно заданию. Имя хоста, версия windows, загрузка процессора и памяти, суммарное свободное место на дисках, список всех процессов, а так же отдельный список отслеживаемых процессов если они запущены, то есть тех что указаны в конфигурации.

Приложение поддерживает очередь отправки (Реализовано через класс `Channel`) и retry при неудачной отправке. Логирование реализовано при помощи библиотеки SeriLog. В решении есть юнит-тесты проверяющие отправку запроса, а так же тесты проверяющие получение системной информации. Логи пишутся в файл и в Windows EventLog. Если приложение запущено в консольном режиме, логи так же пишутся в консоль.
